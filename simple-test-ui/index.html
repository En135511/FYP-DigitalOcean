Guided Support

            
How to use BrailleAI Test Lab

            

              Use this panel whenever you need a fast walkthrough of the main features, expected inputs, and basic troubleshooting.
            

          

          
        


        

          

            Quick Start
            
Get a result in four steps

            
    
              
  1. Open Settings, choose Student or Admin mode, then confirm startup and accessibility options.
    2. 
              
  2. Choose the Braille table that matches the language or grading you want to test.
    3. 
              
  3. Pick a mode: Text for direct text, Image for uploaded Braille photos, or Camera for live capture.
    4. 
              
  4. Press Send to translate, then use response actions to copy output instantly or download the available format (for example BRF in Text to Braille mode).
    5. 
            

          


          

            Modes
            
What each feature does

            
    
              
  • Text: best for typing Braille text or testing translation output quickly.
    • 
              
  • Image: upload a saved photo of embossed Braille for detection and translation.
    • 
              
  • Camera: capture a fresh image directly from your device when supported.
    • 
            

          


          

            Useful Controls
            
Tools around the workspace

            
    
              
  • Service Status: confirms whether the Braille API is reachable.
    • 
              
  • Settings: manages theme, font size, TTS, startup and accessibility.
    • 
              
  • Application Mode: Student mode keeps settings simple, while Admin mode unlocks advanced controls.
    • 
              
  • Advanced Controls (Admin): table overrides, key remapping, repeat hotkey, and tone/extra speech options.
    • 
              
  • Response Actions: use Copy for quick reuse, then click the download icon to choose a file format.
    • 
              
  • Translation Warnings: the assistant warns when input is mixed, ambiguous, or may not translate reliably.
    • 
              
  • Quality Review Flags: image responses now include uncertain-cell counts so you can manually review risky transcriptions.
    • 
              
  • Adaptive Page Pass: full-page uploads are re-checked with segmented processing to recover missed Braille dots.
    • 
              
  • Raw Response Log: shows the latest endpoint activity for debugging.
    • 
            

          


          

            Troubleshooting
            
If something is not working

            
    
              
  • Use Service Status first when translations fail or tables do not load.
    • 
              
  • For camera access, run the app on localhost or over HTTPS and allow browser permissions.
    • 
              
  • If Copy Output fails, allow clipboard permissions in the browser and try again.
    • 
              
  • If uploads fail, capture a flat page with even light, no shadows, minimal tilt, and no pen marks over Braille dots.
    • 
              
  • If review is recommended, verify uncertain cells before generating final exam or embosser files.
    •
